Multi-Tenant Strategy

Now: Single DB (BdStockOMS) with BrokerageHouseId row-level isolation on every table. Every query double-verified against JWT claim. Now: IFIXConnector interface with SimulatedFIXConnector default. All FIX message logic coded against the interface. Live: Drop in QuickFIXConnector implementing the same interface — configure SenderCompID, TargetCompID, host, port via Admin Settings. Zero code changes to order flow.

05

Core Design Principles — Every Line of Code

🔗 Everything Connected

Every widget shows live data from SignalR where possible. Every table has a REST fallback for initial load. If market data unavailable: demo data. No empty states, no loading spinners without fallback content.

⚡ Optimistic UI First

Every action (place order, cancel, modify) updates the UI immediately, confirms from server, reverts on error. P95 API response < 50ms for market data. Redis cache first, DB fallback. Zero unnecessary re-renders.

🔐 Multi-Tenant by Default

BrokerageHouseId from JWT is the first guard on every query. No service method exists without tenant isolation. SuperAdmin bypasses cleanly. Per-tenant feature flags for exchange config, FIX config, logo, branding.

🎛️ Micro-Level Control

Every element: RMS limits per-stock, per-sector, per-board, per-category, per-user, per-BO-group, per-basket, per-branch, per-broker. Column show/hide, resize, reorder persisted. Every table has Excel download. Every form has keyboard shortcuts.

🌍 i18n Everything

All UI strings go through t('key') from Day 79 onward. English keys are added as each component is built. Bengali and Chinese translations added Day 112. Toggle in header persists to localStorage.

🧪 Test Everything

Every new service: xUnit tests same day. Every new frontend component: Vitest test same day (from Day 101). Every critical journey: Playwright E2E. Target: 1,800+ tests by Day 130, 0 failures, always.

Non-Negotiable Development Rules

📁 One Branch Per Day

git checkout -b day-{N}-{slug} at start of every day. Push before sleep. Never commit directly to main. PR merge only when tests pass.

🧪 Tests Same Day

Every new service: xUnit tests in same commit. dotnet test must pass before commit. Zero tolerance for failing tests. 0 failures always means always.

🔑 Tenant Isolation Always

Every DB query: Where(x => x.BrokerageHouseId == tenantId). tenantId comes from JWT only — never from request body. If you see a service method without this: fix it same day.

🌐 i18n From Day 100

From Day 100 onward, every new UI string uses t('key.subkey'). Add the English, Bengali, and Chinese values same day. Never hardcode text in components.

⚡ Live Data or Fallback

Every widget attempts live data. If API fails or market is closed: fallback to demo data (never empty). Loading skeleton while fetching. Error state with retry button if demo fails too.

🐱 Git Bash Heredocs

Frontend files: cat > file << 'ENDOFFILE'. C# files with !: use python -c with list-of-strings. Run set +H before any bash with ! chars. Stop API before build if locked.

⚠️ Critical Technical Rules

OrderType = Buy | Sell (the side). OrderCategory = Market | Limit | MarketAtBest (the type). OrderStatus.Filled not Executed. Hub: StockPriceHub, signal: TradeExecuted. Auth store: @/store/authStore. DTO namespace: BdStockOMS.API.Models.Admin. DB: BdStockOMS (single DB, row-level isolation). AppDbContext is at BdStockOMS.API/Data/AppDbContext.cs not Models/.
